## 这是什么？

**对称加密渗出**（T1048.002）是 **通过替代协议渗漏**（T1048）的一个具体变体，属于 **渗漏** 阶段的攻击技术。

简单来说，就像把偷到的东西用明信片寄出去，谁都能看到内容。

### 具体怎么理解？

用同一把钥匙加解密数据后传输，虽然被看到但打不开

攻击者使用这种技术时，通常是在渗漏阶段，想要达到特定的攻击目标。与父技术 T1048 相比，T1048.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["攻击者入侵系统获取敏感数据"] --> B["生成对称加密密钥（如AES-256）"]
    B --> C["使用密钥加密窃取的数据文件"]
    C --> D["通过替代协议（如HTTPS/ICMP）发送密文"]
    D --> E["外部服务器接收并存储加密数据"]
    E --> F["通过安全通道传输密钥完成解密"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获取敏感数据** - 攻击者在目标系统中定位并读取需要窃取的文件和数据库内容
2. **生成对称密钥** - 创建AES-256等对称加密算法的随机密钥，用于加密待传输数据
3. **加密数据** - 使用密钥对敏感数据进行加密，生成无法直接读取的密文
4. **替代协议传输** - 通过HTTPS、ICMP隧道或DNS隧道等替代协议将密文发送到攻击者控制的服务器
5. **密钥单独传输** - 将加密密钥通过另一个安全通道（如不同的C2通道）传输给攻击者，实现密钥与密文分离

### 应用层检测

**用人话说：** 攻击者使用对称加密算法（如AES-256）对窃取的数据进行加密后再传输，加密密钥和密文通过不同通道分别发送——即使密文被拦截，没有密钥也无法解密。这相当于把赃物锁进密码箱，密码通过另一个秘密渠道告知同伙。与勒索软件不同，对称加密渗出中加密的目的是保护传输中的数据不被安全设备检查出来，而不是为了勒索。如果发现系统出现大规模的文件加密操作，但加密后的文件紧接着通过网络发送出去而不是留在本地，需要警惕是加密渗出而非勒索软件。
