## 这是什么？

**非对称加密渗出**（T1048.001）是 **通过替代协议渗漏**（T1048）的一个具体变体，属于 **渗漏** 阶段的攻击技术。

简单来说，就像把偷到的东西用明信片寄出去，谁都能看到内容。

### 具体怎么理解？

用公钥加密，只有攻击者的私钥能解密，更安全

攻击者使用这种技术时，通常是在渗漏阶段，想要达到特定的攻击目标。与父技术 T1048 相比，T1048.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["攻击者预先生成RSA密钥对"] --> B["将公钥植入目标系统"]
    B --> C["获取目标敏感数据"]
    C --> D["使用公钥加密数据（仅私钥可解密）"]
    D --> E["通过网络发送密文到外部服务器"]
    E --> F["攻击者用私钥解密获取原始数据"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **生成RSA密钥对** - 攻击者在攻击前生成RSA-2048或RSA-4096密钥对，将公钥植入到恶意工具中
2. **植入公钥** - 通过初始入侵将包含公钥的恶意程序部署到目标系统中
3. **获取敏感数据** - 在目标系统中定位和读取需要窃取的高价值数据文件
4. **公钥加密** - 使用植入的公钥对数据进行非对称加密，生成的密文只有攻击者的私钥能解密
5. **传输与解密** - 通过网络将密文发送到外部服务器，攻击者使用私钥完成解密，获取原始数据

```bash

```

### 应用层检测

**用人话说：** 攻击者在窃取数据时，先用目标系统上的公钥对数据进行非对称加密（如RSA），再通过网络发送出去——即使传输过程中被拦截，没有攻击者的私钥也无法解密数据。这相当于把赃物装进一个只有自己才有钥匙的保险箱里寄出去。与勒索软件加密不同，这里的加密目的不是破坏数据，而是保护窃取的通道安全，防止数据在传输途中被他人截获。如果发现系统上有非预期的OpenSSL、GnuPG等加密工具执行，且加密操作后紧跟着网络传输行为，可能是攻击者在加密打包准备窃取的数据。

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
```
