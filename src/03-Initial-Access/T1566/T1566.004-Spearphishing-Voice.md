# 鱼叉式钓鱼语音（Vishing） (T1566.004)

## 一句话通俗理解

> **通过语音电话进行社会工程学攻击**

## 30秒速查卡

| 为什么危险？ | 电话钓鱼完全绕过所有技术防护，直接攻击人的信任心理，而且没有"邮件网关"能拦截 |

## 这是什么？

**鱼叉式钓鱼语音（Vishing）**（T1566.004）是 **钓鱼**（T1566）的一个具体变体，属于 **初始访问** 阶段的攻击技术。

简单来说，就像冒充银行客服给你发短信，骗你输入密码。

### 具体怎么理解？

通过语音电话进行社会工程学攻击

攻击者使用这种技术时，通常是在初始访问阶段，想要达到特定的攻击目标。与父技术 T1566 相比，T1566.004 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["收集目标个人信息<br/>OSINT获取目标姓名、职位<br/>联系方式和工作习惯"] --> B["伪造来电号码<br/>使用VoIP技术伪造<br/>为内部号码或可信号码"]
    B --> C["冒充IT支持/供应商<br/>编造紧急技术问题<br/>制造心理压力"] --> D["诱导提供敏感信息<br/>套取系统密码/MFA验证码<br/>或远程桌面访问权限"]
    D --> E["利用窃取信息直接登录<br/>使用获取的凭证登录<br/>企业系统实现初始访问"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **收集目标个人信息** - 攻击者通过 LinkedIn、公司组织架构、新闻报道等 OSINT 手段收集目标姓名、职位、联系方式、直属上级和 IT 支持团队信息
2. **伪造来电号码** - 使用 VoIP 服务（如 Twilio、SpoofCard）伪造来电显示，使其显示为内部 IT 支持热线、银行客服或政府机构号码
3. **冒充可信角色** - 电话接通后冒充 IT 支持人员、供应商项目经理或 CEO 助理，编造紧急账户恢复、系统升级或业务验证场景制造心理压力
4. **诱导提供敏感信息** - 利用紧迫感和权威感，诱导目标提供系统密码、MFA 验证码、远程桌面访问权限或 VPN 接入信息
5. **直接登录实现初始访问** - 使用通过电话获取的凭证或远程访问权限，直接登录目标企业系统，实现初始访问

### 应用层检测

**用人话说：** 攻击者直接打电话冒充公司IT支持或银行客服——"您好，我是IT运维张工，检测到您的电脑中了病毒，需要您协助处理"——引导受害者安装AnyDesk/TeamViewer等远程控制软件，或骗取VPN OTP验证码。高级Vishing攻击会使用AI语音克隆模拟老板的声音来下达指令。检测上，关注用户在接到异常电话后短时间内安装了远程桌面软件（AnyDesk、TeamViewer、Splashtop）或启动了Windows快速助手，以及新安装的远程软件发起了对外部的出站连接——这些组合行为是Vishing攻击的典型特征。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
