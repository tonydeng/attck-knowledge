# 套接字过滤器 (T1205.002)

## 一句话通俗理解

> **后门在底层设置过滤器捕获符合特征的网络包，匹配后触发激活**

## 这是什么？

**套接字过滤器**（T1205.002）是 **流量信号**（T1205）的一个具体变体，属于 **命令与控制** 阶段的攻击技术。



> 🔐 **打个比方**：就像在门上装了一把只有你知道暗号才能打开的锁——攻击者使用密码学方法隐藏和验证C2通信的合法性。

### 具体怎么理解？

后门在底层设置过滤器捕获符合特征的网络包，匹配后触发激活

攻击者使用这种技术时，通常是在命令与控制阶段，想要达到特定的攻击目标。与父技术 T1205 相比，T1205.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **命令与控制** 阶段使用 套接字过滤器 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["在内核层设置BPF过滤器"] --> B["过滤器匹配特定网络包特征"]
    B --> C["匹配成功触发后门激活"]
    C --> D["建立隐蔽C2通信通道"]
    D --> E["通过过滤通道传输指令"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **在内核层设置BPF过滤器** - 使用Berkeley包过滤器在底层设置网络包捕获规则
2. **过滤器匹配特定网络包特征** - 配置过滤器识别包含特定Magic Bytes的网络包
3. **匹配成功触发后门激活** - 当收到匹配特征的网络包时，后门自动激活（关键步骤）
4. **建立隐蔽C2通信通道** - 后门打开底层Socket建立C2数据传输通道
5. **通过过滤通道传输指令** - 利用过滤后的隐蔽通道传输加密的C2指令


### 应用层检测

**用人话说：** 套接字过滤器（Socket Filters）后门利用操作系统底层的网络包捕获机制来"监听暗号"。攻击者在内核层设置一个BPF（Berkeley Packet Filter）过滤器，监听所有经过网卡的网络包——当某个特定特征的包出现（如TCP SYN到特定端口、IP包中包含特定字节序列），过滤器就触发后门激活C2通道。这比端口敲门更隐蔽，因为后门进程根本不在任何端口上listen，用netstat -an看不到任何异常端口。检测方法：监控进程创建原始套接字（AF_PACKET、SOCK_RAW）的行为、加载BPF过滤器的操作（Linux的setsockopt SO_ATTACH_FILTER）、以及Windows下的WinPcap/Npcap驱动加载。正常企业应用极少使用原始套接字。
