# 端口敲门 (T1205.001)

## 一句话通俗理解

> **攻击者按顺序访问特定端口来"敲门"，后门检测到正确序列后激活**

## 30秒速查卡

| 维度 | 你需要知道的 |

| 这是什么？ | 攻击者按特定顺序访问一系列端口（如先80再443再8080），后门检测到正确的"敲门"序列后才激活C2通道 |
| 为什么危险？ | 敲门流量看起来就是普通的端口扫描或连接尝试，没有正确序列就看不到后门，传统安全设备无法检测 |

⭐⭐⭐ 高级 - 需要深入的技术知识和实践

## 这是什么？

**端口敲门**（T1205.001）是 **流量信号**（T1205）的一个具体变体，属于 **命令与控制** 阶段的攻击技术。



> 🚪 **打个比方**：就像在防火墙规则里留了一个只有你知道的"后门"端口——攻击者在系统中设置特定的触发条件（如端口敲击），在满足条件时才开放C2通道。

### 具体怎么理解？

攻击者按顺序访问特定端口来"敲门"，后门检测到正确序列后激活

攻击者使用这种技术时，通常是在命令与控制阶段，想要达到特定的攻击目标。与父技术 T1205 相比，T1205.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **命令与控制** 阶段使用 端口敲门 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["在目标系统植入端口敲门后门"] --> B["所有C2端口默认关闭"]
    B --> C["攻击者按序列访问特定端口"]
    C --> D["后门检测到正确敲门序列"]
    D --> E["后门激活开放C2通信端口"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **在目标系统植入端口敲门后门** - 安装支持端口敲门机制的恶意后门程序
2. **所有C2端口默认关闭** - 后门监听特定端口序列，但对外表现为所有端口关闭
3. **攻击者按序列访问特定端口** - 攻击者依次连接预定义的端口序列（如7000→8000→9000）
4. **后门检测到正确敲门序列** - 后门验证敲门序列正确后激活C2通道（关键步骤）
5. **后门激活开放C2通信端口** - 动态开放指定端口，允许攻击者建立C2连接

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 端口敲门（Port Knocking）就像秘密组织的接头暗号——攻击者按特定顺序访问一组端口（如先发一个SYN到端口80、然后到22、再到8080），防火墙检测到这一"敲门序列"后临时开放一个隐藏的服务端口供攻击者连接。被控机器上的后门程序在监听网络层数据包，如果收到正确的敲门序列就激活C2通道开始通信。这种技术让后门在"安静"状态下对任何端口扫描都不响应，看起来像是一台干净的机器。检测方法：在防火墙日志中搜索短时间内来自同一IP的多个有序端口访问（特别是非服务端口），然后突然切换到一个之前被拒绝的新端口进行完整TCP连接。正常端口扫描通常是无序的或者按端口号顺序扫的。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
