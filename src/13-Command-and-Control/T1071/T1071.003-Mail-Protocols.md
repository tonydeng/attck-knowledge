# 邮件协议 (T1071.003)

## 一句话通俗理解

> **用 SMTP/IMAP/POP3 收发邮件传指令，适合邮件流量大的单位**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者用SMTP发邮件传出数据、用IMAP/POP3收邮件获取指令，把C2通信伪装成正常的收发邮件 |
| 为什么危险？ | 任何企业都有邮件服务器，SMTP/IMAP流量是日常办公必备，安全设备不会拦截邮件协议流量 |
| 谁需要关心？ | 邮件管理员、SOC分析师、安全运营团队 |
| 你的第一步防御 | 监控非邮件客户端进程（如PowerShell、cmd）连接SMTP端口（25/587/465）的行为 |
| 如果只做一件事 | 建立邮件服务器出站连接基线，重点关注使用非常规客户端发送大量邮件的主机 |

## 难度等级

⭐⭐ 中级 - 需要一定的技术基础和经验

## 这是什么？

**邮件协议**（T1071.003）是 **应用层协议**（T1071）的一个具体变体，属于 **命令与控制** 阶段的攻击技术。



> 📡 **打个比方**：就像在广播电台的正常节目中插入暗语——攻击者利用常见的网络协议（如MQTT、CoAP）传输C2指令。

### 具体怎么理解？

用 SMTP/IMAP/POP3 收发邮件传指令，适合邮件流量大的单位

攻击者使用这种技术时，通常是在命令与控制阶段，想要达到特定的攻击目标。与父技术 T1071 相比，T1071.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **命令与控制** 阶段使用 邮件协议 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["注册专用C2邮箱账户"] --> B["木马通过SMTP发送窃取数据"]
    B --> C["攻击者通过邮件发送加密指令"]
    C --> D["木马通过IMAP收取指令邮件"]
    D --> D2["解密执行并回复结果"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **注册专用C2邮箱账户** - 在公共邮箱服务商处注册用于C2通信的专用邮箱
2. **木马通过SMTP发送窃取数据** - 恶意载荷通过SMTP协议将窃取的数据作为邮件正文或附件发出
3. **攻击者通过邮件发送加密指令** - 攻击者向专用邮箱发送包含加密攻击指令的邮件（关键步骤）
4. **木马通过IMAP收取指令邮件** - 木马通过IMAP协议连接邮箱服务器，读取新邮件
5. **解密执行并回复结果** - 解密邮件中的指令并执行，将结果通过新邮件回复

### 检测思路

检测 邮件协议 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输

```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 攻击者利用邮件协议（SMTP/IMAP/POP3）传C2指令，就像把情报藏在平信里寄出去。木马可以在被控机器上配置一个邮件客户端，把窃取的数据以邮件附件发送到攻击者控制的邮箱账号；反过来，攻击者可以给这个邮箱发一封标题为"会议纪要"的邮件，里面其实加密了下一步指令。某些APT组织会注册免费邮箱（Gmail、Yahoo、ProtonMail）作为C2中继站，因为邮件流量在企业网络中通常不会被拦截。检测方法：监控服务器上非用户操作的邮件客户端进程、异常时间段的大量邮件收发、以及邮件附件中包含加密的zip或可执行文件。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
