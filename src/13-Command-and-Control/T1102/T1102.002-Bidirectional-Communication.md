# 双向通信 (T1102.002)

## 一句话通俗理解

> **攻击者和受害者通过Web服务的API实现双向C2通信**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者利用GitHub、Google Drive、Twitter等Web服务的API实现C2指令的上传和下载，形成双向通信 |
| 为什么危险？ | API流量与正常应用流量无法区分，CDN保护让这些域名难以封锁，企业普遍允许访问 |
| 谁需要关心？ | SOC分析师、云安全工程师、API安全团队 |
| 你的第一步防御 | 监控异常的Web服务API调用模式，尤其是非浏览器进程的OAuth认证行为 |
| 如果只做一件事 | 审计GitHub/Google Drive API调用日志，识别单账户高频次的读写操作 |

## 难度等级

⭐⭐ 中级 - 需要一定的技术基础和经验

## 这是什么？

**双向通信**（T1102.002）是 **Web服务**（T1102）的一个具体变体，属于 **命令与控制** 阶段的攻击技术。



> ☁️ **打个比方**：就像把指令藏在共享云盘的文件里——攻击者利用云存储服务（如Google Drive、OneDrive）同步C2指令和数据。

### 具体怎么理解？

攻击者和受害者通过Web服务的API实现双向C2通信

攻击者使用这种技术时，通常是在命令与控制阶段，想要达到特定的攻击目标。与父技术 T1102 相比，T1102.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **命令与控制** 阶段使用 双向通信 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["利用GitHub/云存储创建仓库"] --> B["木马通过API上传窃取数据"]
    B --> C["攻击者通过API下发指令"]
    C --> D["木马读取仓库中的新指令"]
    D --> E["双向通过合法API交换数据"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **利用GitHub/云存储创建仓库** - 在合法云服务上创建用于C2通信的代码仓库或存储桶
2. **木马通过API上传窃取数据** - 恶意载荷使用API将窃取的数据上传到仓库的指定位置
3. **攻击者通过API下发指令** - 攻击者将加密指令写入仓库文件，通过API发布更新（关键步骤）
4. **木马读取仓库中的新指令** - 载荷定期检查仓库变更，获取新指令并执行
5. **双向通过合法API交换数据** - 利用合法云服务API实现双向C2通信，流量与正常用户混合

## 检测建议

### 检测思路

检测 双向通信 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输
# 根据具体协议和端口设置检测规则
```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 双向通信模式让攻击者通过同一个Web服务API同时收发数据——被控机器向GitHub Issues、Google Drive或Notion API提交窃取的数据作为"上传"，又从同一个服务的另一处读取加密指令作为"下载"。这就像两个人用同一块公告板交换消息，一人贴一人读。这种方式的隐蔽性在于流量完全混入正常的API调用中，如果攻击者使用的API Token来自合法账号，很难区分是正常用户还是木马。检测思路：某台主机用同一个API Token频繁读写不同的资源（如创建Issue后马上读取项目Wiki），且读写间隔非常有规律，这不太像真人操作。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
