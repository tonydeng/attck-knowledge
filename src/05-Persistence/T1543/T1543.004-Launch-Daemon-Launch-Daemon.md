## 这是什么？

**启动守护进程**（T1543.004）是 **创建或修改系统进程**（T1543）的一个具体变体，属于 **持久化** 阶段的攻击技术。



> 🌐 **打个比方**：就像在Windows的IIS服务器管理器中添加了一个恶意网站——攻击者创建或修改Windows服务，使其在特定的运行级别下自动启动。

### 具体怎么理解？

macOS系统级守护进程

攻击者使用这种技术时，通常是在持久化阶段，想要达到特定的攻击目标。与父技术 T1543 相比，T1543.004 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["获得macOS root权限"] --> B["创建恶意Launch Daemon plist"]
    B --> C["将plist放置在/Library/LaunchDaemons/"]
    C --> D["设置KeepAlive为true确保常驻"]
    D --> E["加载LaunchDaemon（系统启动时加载）"]
    D --> F["恶意守护进程以root权限自动运行"]
    style B fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获得macOS root权限** - 通过漏洞利用或sudo提权获得Mac系统的root权限
2. **创建恶意plist** - 编写launchd.plist，设置Label、ProgramArguments和KeepAlive
3. **放置到LaunchDaemons** - 将plist复制到/Library/LaunchDaemons/（系统级自动启动）
4. **设置KeepAlive** - 将KeepAlive设置为true，确保恶意进程崩溃后自动重启
5. **加载守护进程** - 使用`launchctl load /Library/LaunchDaemons/malicious.plist`加载
6. **root权限自动运行** - 系统启动时以root权限自动启动恶意守护进程，持续运行

### 应用层检测


**用人话说：** 就像在Mac系统启动时自动加载了一个内核扩展——Launch Daemon是macOS系统级别的守护进程，通过`/Library/LaunchDaemons/`目录下的.plist文件定义。与Launch Agent不同，Launch Daemon在系统启动时就会加载，不需要任何用户登录，以root权限运行。攻击者创建一个plist文件，设置`KeepAlive=true`让恶意进程崩溃后自动重启。因为需要root权限才能写入`/Library/LaunchDaemons/`，攻击者通常先提权再部署。这也是为什么macOS恶意软件经常要求输入管理员密码的原因之一。
