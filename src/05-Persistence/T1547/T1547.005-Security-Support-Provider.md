## 这是什么？

**安全支持提供者**（T1547.005）是 **启动或登录自动执行**（T1547）的一个具体变体，属于 **持久化** 阶段的攻击技术。

简单来说，就像在电脑启动时自动运行的定时炸弹。

### 具体怎么理解？

认证相关的安全DLL

攻击者使用这种技术时，通常是在持久化阶段，想要达到特定的攻击目标。与父技术 T1547 相比，T1547.005 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["获得Windows SYSTEM权限"] --> B["准备恶意SSP DLL"]
    B --> C["将DLL复制到System32目录"]
    C --> D["修改注册表Security Packages"]
    D --> E["系统启动时LSASS加载SSP"]
    D --> F["恶意DLL捕获所有登录凭据"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获得SYSTEM权限** - 需要系统权限才能修改LSASS加载的安全包配置
2. **准备恶意SSP** - 编写实现Security Support Provider接口的恶意DLL（导出SpInitialize等函数）
3. **复制到System32** - 将恶意DLL复制到C:\Windows\System32目录中
4. **修改注册表** - 在`HKLM\System\CurrentControlSet\Control\Lsa\Security Packages`中添加SSP名称
5. **系统启动LSASS加载** - Windows启动时LSASS加载所有注册的Security Support Provider
6. **捕获登录凭据** - 恶意SSP在LSASS进程中监听所有认证事件，记录明文密码和NTLM哈希

- /var/log/syslog：系统日志


### 应用层检测


**用人话说：** 就像给银行的ATM机加装了一个键盘记录器——SSP（安全支持提供者）是Windows认证框架的一部分，LSASS进程在启动时会加载所有注册的SSP DLL。攻击者在`HKLM\SYSTEM\CurrentControlSet\Control\Lsa\Security Packages`中添加恶意DLL名称，系统重启后LSASS就会加载它。这个DLL能监听到所有用户的明文密码和NTLM哈希，因为LSASS处理的就是认证数据。mimikatz的`misc::memssp`命令就是利用这个原理，一条命令就能让LSASS开始记录凭据。
