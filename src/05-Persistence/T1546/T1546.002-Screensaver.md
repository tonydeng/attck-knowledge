## 这是什么？

**屏幕保护程序**（T1546.002）是 **事件触发执行**（T1546）的一个具体变体，属于 **持久化** 阶段的攻击技术。



> 🔧 **打个比方**：就像给Windows的屏幕保护程序换成了恶意程序——攻击者修改屏幕保护程序设置，屏保启动时自动执行恶意代码。

### 具体怎么理解？

替换屏幕保护程序为恶意程序

攻击者使用这种技术时，通常是在持久化阶段，想要达到特定的攻击目标。与父技术 T1546 相比，T1546.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["获得Windows系统访问权限"] --> B["将恶意程序重命名为.scr文件"]
    B --> C["替换或修改屏幕保护程序注册表"]
    C --> D["在HKCU\Control Panel\Desktop中设置SCRNSAVE.EXE"]
    D --> E["系统空闲等待设定时间（默认15分钟）"]
    D --> F["屏幕保护启动时恶意程序自动运行"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获得Windows访问权限** - 通过漏洞利用或凭据窃取获得目标系统的注册表写入权限
2. **准备恶意.scr文件** - 将恶意后门程序重命名为.scr后缀（Windows屏幕保护程序格式）
3. **修改注册表** - 修改HKEY_CURRENT_USER\Control Panel\Desktop中的SCRNSAVE.EXE键值
4. **设置启动路径** - 将SCRNSAVE.EXE的值设为恶意.scr文件的完整路径
5. **等待系统空闲** - 系统在设定的空闲时间（默认为15分钟）后自动激活屏幕保护
6. **自动执行恶意程序** - 屏幕保护启动时，恶意.scr文件以用户权限自动运行

### 应用层检测


**用人话说：** 就像你把手机设为"空闲5分钟后自动打开抖音"——但实际打开的是间谍软件。Windows的屏幕保护程序是一个`.scr`（本质上是PE可执行文件），在注册表`HKCU\Control Panel\Desktop`的`ScreenSaveActive`和`SCRNSAVE.EXE`键值中配置。攻击者把自己的恶意程序重命名为`.scr`文件替换掉正常屏保，或者直接修改注册表中的路径。当电脑空闲指定时间后，这个恶意屏保就会以当前用户权限自动启动。因为屏保是系统正常功能，即使用户发现进程列表里有它也不会起疑心。
