## 这是什么？

**内部网页篡改**（T1491.001）是 **网页篡改**（T1491）的一个具体变体，属于 **影响** 阶段的攻击技术。



> 🏚️ **打个比方**：就像在别人的公司网站上喷漆涂鸦——攻击者篡改网站首页，展示政治宣言或炫耀入侵成果。

### 具体怎么理解？

篡改公司内部网站的内容，如内部门户、内部通知页面

攻击者使用这种技术时，通常是在影响阶段，想要达到特定的攻击目标。与父技术 T1491 相比，T1491.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["攻击者渗透进入企业内网"] --> B["定位内部门户服务器"]
    B --> C["获取内部门户CMS后台权限"]
    C --> D["修改内部通知/门户页面内容"]
    D --> E["内部员工访问时看到被篡改内容"]
    E --> F["造成内部恐慌和信息混乱"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **渗透内网** - 攻击者通过VPN漏洞、鱼叉式钓鱼或内部人员协助获得企业内网访问权限
2. **定位门户服务器** - 扫描内网发现SharePoint、Confluence或企业自建的内部门户网站
3. **获取后台权限** - 利用弱密码、SSO令牌劫持或CMS漏洞获得后台管理权限
4. **篡改页面内容** - 修改内部通知页面、公告栏、企业文化页面的内容，植入虚假信息或攻击声明
5. **内部影响** - 内部员工打开内部门户时看到被篡改的内容，可能误信虚假通知导致恐慌或执行错误操作

**方法**：监控网络流量中的异常模式


### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 这条规则在检测内部网页篡改——攻击者改的不是对外网站，而是公司内部的门户页面（如SharePoint、Confluence、内部公告栏）。攻击者通过VPN钓鱼或内网渗透获得内部系统权限后，替换内部通知页面内容，可能发布虚假通知（如"公司放假"）、散布恐慌信息或植入恶意链接。检测的关键信号是：内部Web服务器在非发布窗口期的文件修改、CMS后台出现异常登录（来源IP非内部办公子网）、或内部员工反馈看到异常页面内容。内部篡改比外部篡改更难被发现——因为外部用户看不到内部页面，往往在攻击者自己公开后才被察觉。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
