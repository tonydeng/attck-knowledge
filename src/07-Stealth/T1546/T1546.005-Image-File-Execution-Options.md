# 映像劫持 (T1546.005)

## 一句话通俗理解

> **修改程序启动路径，在程序启动前执行恶意代码**

## 这是什么？

**映像劫持**（T1546.005）是 **事件触发执行**（T1546）的一个具体变体，属于 **隐蔽** 阶段的攻击技术。

> 📚 **打个比方**：就像在别人家门外贴了一张假纸条"请先按隔壁门铃"——攻击者修改IFEO注册表，当用户启动正常程序（如浏览器）时，系统会先执行攻击者的恶意代码，然后再启动真正的程序，用户感觉一切正常。

### 具体怎么理解？

修改程序启动路径，在程序启动前执行恶意代码

攻击者使用这种技术时，通常是在隐蔽阶段，想要达到特定的攻击目标。与父技术 T1546 相比，T1546.005 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **隐蔽** 阶段使用 映像劫持 技术，以下是典型的攻击步骤：






```mermaid

    E["恶意代码借合法程序触发"]

    A -->|"添加注册"| B
    B -->|"设Debugger"| C

```

**步骤详解：**

> **步骤1**：选择一个系统信任的目标程序

> **步骤2**：在注册表Image File Execution Options中添加项

> **步骤3**：设置Debugger值指向恶意程序

> **步骤4**：启动目标程序时系统先执行Debugger指定的恶意程序
