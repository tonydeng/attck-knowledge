## 这是什么？

**额外电子邮件委派权限**（T1098.002）是 **账户操纵**（T1098）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

> 📚 **打个比方**：就像在公司的邮件系统中悄悄把自己设置为CEO的"助理"——从此CEO的所有邮件都会自动抄送给你，而CEO本人完全不知道自己的收件箱多了一个"旁观者"。

### 具体怎么理解？

给自己授予访问他人邮箱的权限

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1098 相比，T1098.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 额外电子邮件委派权限 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取邮箱账户<br/>管理员权限"] --> B["使用Exchange或Graph API<br/>为目标邮箱添加委派权限"]
    B --> C["将攻击者账户添加为<br/>FullAccess和SendAs权限"]
    C --> D["通过Outlook或OWA<br/>直接访问目标邮箱"]
    D --> E["读取敏感邮件<br/>窃取商业机密"]
    E --> F["以目标身份发送邮件<br/>进行社会工程攻击"]

    style B fill:#ff6b6b,stroke:#333,stroke-width:2px
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **权限授予**：通过 Exchange PowerShell `Add-MailboxPermission` 或 Graph API `addMailboxPermission` 添加 FullAccess 权限
2. **发送权限**：额外配置 `Send-As` 权限，允许以目标邮箱地址发送邮件
3. **隐蔽访问**：通过 Outlook 配置文件或 OWA Web 界面直接读取目标邮件，目标用户不会收到通知
4. **钓鱼利用**：以目标身份发送钓鱼邮件给其他员工，利用信任关系扩大攻击范围

**用人话说：** 邮箱委托权限允许一个用户访问另一个用户的邮箱，包括读取、发送和管理邮件。攻击者在获得Exchange管理员或邮箱所有者权限后，给恶意账户授予FullAccess和SendAs委托权限。这样攻击者可以持续监控高管的邮件往来，或者冒充高管发送钓鱼邮件给内部员工。这就像给攻击者发了一把目标邮箱的后备钥匙——他可以随时查看和发送邮件而不被察觉。
