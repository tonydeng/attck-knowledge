## 这是什么？

**云策略修改**（T1484.003）是 **域或租户策略修改**（T1484）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

> 📚 **打个比方**：就像偷偷修改了写字楼的安保值班表——把本该严格查验证件的入口改成了"免检通道"，从此所有人员和物品都能畅通无阻地进入大楼。

### 具体怎么理解？

修改 Azure AD/Office 365 的条件访问和安全策略

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1484 相比，T1484.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 云策略修改 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取Azure AD<br/>全局管理员权限"] --> B["访问Azure Portal<br/>或使用Graph API"]
    B --> C["修改条件访问策略<br/>禁用MFA和IP限制"]
    C --> D["修改权限提升策略<br/>允许普通用户注册管理角色"]
    D --> E["利用放宽的策略<br/>以任意用户身份登录"]
    E --> F["通过自助提权<br/>获取Global Admin权限"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **策略访问**：通过 Azure Portal 或 Microsoft Graph API（`/identity/conditionalAccess/policies`）访问条件访问策略
2. **MFA 绕过**：修改或禁用条件访问策略中的多因素认证要求，使弱密码即可登录
3. **提权通道开放**：修改 Privileged Identity Management（PIM）策略，放宽角色激活的审批要求
4. **权限提升**：利用放宽的策略，通过 Azure AD 的自助服务门户（My Access）激活高权限角色（如 Global Administrator）

**用人话说：** 云策略修改是指攻击者篡改Azure AD或Office 365中的条件访问策略、安全默认值或身份保护策略。例如，创建一个条件访问策略允许来自"所有位置"和"所有设备"的访问，或者禁用MFA要求。使用`New-AzureADMSConditionalAccessPolicy`或Azure门户进行操作。这就像拆掉了云环境中的所有安防摄像头和门禁——让所有安全限制对攻击者失效，同时不影响正常用户的感知。

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
```
