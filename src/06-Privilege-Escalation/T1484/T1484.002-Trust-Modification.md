## 这是什么？

**信任修改**（T1484.002）是 **域或租户策略修改**（T1484）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

> 📚 **打个比方**：就像在两个原本互不信任的小区之间伪造了一份"友好社区协议"——拿着A小区的门禁卡竟然能刷开B小区的大门，两个小区的保安还都觉得"这是正常授权"。

### 具体怎么理解？

修改 AD 信任关系或联合身份验证配置，伪造身份令牌

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1484 相比，T1484.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 信任修改 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取域管理员权限<br/>修改AD信任关系"] --> B["修改域间信任的<br/>SID筛选和鉴别设置"]
    B --> C["禁用SID筛选<br/>使跨域SID验证失效"]
    C --> D["利用信任关系<br/>伪造跨域SID令牌"]
    D --> E["以受信任域的管理员<br/>身份访问目标域资源"]
    E --> F["通过伪造的SID<br/>实现跨域权限提升"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style E fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **信任关系枚举**：使用 `nltest /domain_trusts` 枚举域间信任关系，识别可利用的跨域信任
2. **SID 筛选禁用**：修改信任属性中的 SID 筛选（SID Filtering）设置，允许伪造的 SID 通过跨域验证
3. **票据伪造**：利用 Kerberos 的 S4U 扩展或 Golden Ticket 技术，构造包含目标域高权限 SID 的 TGT 票据
4. **跨域访问**：使用伪造的票据以目标域管理员身份访问资源，实现从低信任域到高信任域的权限提升


**用人话说：** 信任修改是指攻击者篡改Active Directory域之间或AD与Azure AD之间的信任关系。攻击者可以在AD FS（联合身份验证服务）中添加一个新的信任关系，使攻击者控制的身份提供者可以签发有效的令牌。使用`Set-ADTrust`或`New-ADFSTrustedRelyingParty`等PowerShell命令操作。这就像在两家公司之间建立了一个假的合作关系——攻击者的系统被当作可信合作伙伴，可以随意访问对方的资源。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```
