## 这是什么？

**通俗解释：**
使用 Linux 的 at 命令创建一次性定时任务



> ⏰ **打个比方**：就像设定一个每天凌晨3点自动运行的闹钟——攻击者使用Windows at命令创建定时任务，在特定时间执行恶意程序。

**技术原理：**
T1053.002 是 计划任务/作业（T1053）的子技术，专注于At (Linux)这一特定方面。攻击者在权限提升阶段，通过At (Linux)来获取目标系统或组织的相关信息，为后续攻击步骤做准备。

**用途与影响：**
- 为后续攻击提供关键信息支撑
- 提高攻击的成功率和精准度
- 降低攻击被发现的概率

## 真实攻击流程

```mermaid
graph TD
    A["攻击者获取root权限<br/>或at命令访问权限"] --> B["执行at命令<br/>at now + 5 minutes"]
    B --> C["通过管道输入<br/>恶意命令内容"]
    C --> D["atq查看任务队列<br/>确认任务已创建"]
    D --> E["系统atd守护进程<br/>在指定时间执行命令"]
    E --> F["恶意代码以root权限<br/>执行反弹Shell"]

    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **任务创建**：使用 `echo "/tmp/evil.sh" | at now + 5 minutes` 或 `at now + 10 minutes << EOF\n/tmp/shell.sh\nEOF` 创建一次性任务
2. **任务验证**：`atq` 查看任务队列，确认任务 ID 和执行时间
3. **atd 执行**：Linux atd 守护进程在指定时间以提交用户权限执行命令
4. **权限继承**：若以 root 身份提交 at 任务，恶意代码以 root 权限运行；任务执行后自动从队列删除



1. **确定目标** - 明确要攻击的组织或个人
2. **选择At (Linux)方法** - 根据目标特点选择最有效的收集方法
3. **执行At (Linux)操作** - 使用相应的工具和技术进行信息收集
4. **整理和分析信息** - 对收集到的数据进行分类和验证
5. **利用信息进行下一步攻击** - 基于收集到的信息制定后续攻击计划

- **时间**: 2023-2024年

### 主机层检测

**Windows事件ID：**

- 事件ID 4688：可疑进程创建
- 事件ID 4104：PowerShell脚本块日志

**Linux日志：**

- 日志文件：`/var/log/syslog`
- 关键字段：可疑命令执行

**用人话说：** Linux的at命令允许用户在指定时间执行一次性任务。攻击者如果获得了低权限shell但该用户在`/etc/at.allow`中或被`/etc/at.deny`放行，可以用`echo "chmod +s /bin/bash" | at now + 1 minute`让系统在1分钟后以root权限执行命令（如果atd服务以root运行）。更常见的是，攻击者通过写权限修改`/etc/cron.allow`或利用配置错误让at任务以更高权限执行。
