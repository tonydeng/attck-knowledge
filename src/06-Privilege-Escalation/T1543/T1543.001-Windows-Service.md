## 这是什么？

**Windows 服务**（T1543.001）是 **创建或修改系统进程**（T1543）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

> 📚 **打个比方**：就像在一栋大楼的电力系统里加装了一个自己的设备——只要大楼通电（系统启动），这个设备就会自动以最高权限运行，而且物业（管理员）检查设备列表时看到的还是一个"官方认证"的服务名称。

### 具体怎么理解？

创建或修改 Windows 服务，使其以 SYSTEM 权限自动运行

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1543 相比，T1543.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 Windows 服务 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取管理员权限<br/>使用sc.exe或PowerShell"] --> B["创建恶意Windows服务<br/>如sc create EvilService binPath"]
    B --> C["设置服务启动类型为<br/>自动启动和SYSTEM运行"]
    C --> D["启动服务<br/>恶意代码以SYSTEM运行"]
    D --> E["服务崩溃或重启后<br/>自动重新执行恶意代码"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **服务创建**：使用 `sc create EvilService binPath= "C:\malware.exe" type= own start= auto` 创建以 SYSTEM 权限运行的服务
2. **权限配置**：设置 `start= auto` 使服务在系统启动时自动运行，设置 `type= own` 确保服务独立进程
3. **立即执行**：使用 `sc start EvilService` 立即启动服务，恶意代码以 SYSTEM 权限运行
4. **持久化保障**：服务注册在 `HKLM\SYSTEM\CurrentControlSet\Services` 中，系统重启后自动恢复

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

**用人话说：** Windows服务（Service）是后台运行的系统进程，通常以SYSTEM或LocalService权限运行。攻击者使用`sc create`命令创建恶意服务：`sc create EvilService binPath= "C:\malware.exe" start= auto`，然后`sc start EvilService`启动它。也可以使用`sc config`修改现有服务的binPath指向恶意程序。因为服务是Windows系统的标准组件，伪装成合法服务名（如"WindowsUpdateService"）的后门很难被发现。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接

iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```
