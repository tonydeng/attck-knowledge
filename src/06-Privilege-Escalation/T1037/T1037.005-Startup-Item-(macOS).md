## 这是什么？

**启动项 (macOS)**（T1037.005）是 **引导或登录初始化脚本**（T1037）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

> 📚 **打个比方**：就像把广告传单塞进别人的"开机自启"文件夹——下次用户开机时，传单（恶意程序）就会自动弹出来，而且因为是在系统启动阶段运行，权限远高于普通程序。

### 具体怎么理解？

在 macOS 的启动项目录中放置恶意程序

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1037 相比，T1037.005 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 启动项 (macOS) 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取写入权限<br/>访问Library目录"] --> B["创建/Library/StartupItems/<br/>evil/目录结构"]
    B --> C["编写恶意可执行文件<br/>和StartupParameters.plist"]
    C --> D["系统启动时BSD子系统<br/>执行StartupItems脚本"]
    D --> E["恶意程序以root权限<br/>在系统启动阶段运行"]
    E --> F["持久化后门<br/>无需用户交互"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style E fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **目录创建**：在 `/Library/StartupItems/` 下创建恶意目录（如 `evil/`），包含可执行文件和配置
2. **参数配置**：编写 `StartupParameters.plist` 定义启动顺序（`OrderPreference`）和所需权限
3. **启动执行**：macOS 的 BSD 子系统在启动过程中检测并执行 StartupItems 中的脚本
4. **root 权限**：StartupItems 以 root 权限运行，早于用户登录阶段，提供早期持久化

**用人话说：** macOS的启动项存放在`/Library/StartupItems/`（系统级）或`~/Library/StartupItems/`（用户级），系统启动时自动执行其中的脚本或程序。攻击者创建一个包含恶意可执行文件的启动项目录，配合一个`StartupParameters.plist`配置文件，使其在系统启动时以root权限运行。这就像在Mac的"开机自启应用列表"里偷偷加了一个伪装成系统服务的后门程序，每次开机自动启动且不会弹出任何提示。
