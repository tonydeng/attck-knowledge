## 这是什么？

**LSASS 驱动**（T1547.008）是 **启动或登录自动启动执行**（T1547）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

简单来说，就像在电脑启动时自动运行的定时炸弹。

### 具体怎么理解？

在 LSASS 中加载恶意驱动程序

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1547 相比，T1547.008 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 LSASS 驱动 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取管理员权限<br/>编写恶意内核驱动"] --> B["将驱动注册为<br/>LSASS相关认证组件"]
    B --> C["系统启动时LSASS<br/>进程加载恶意驱动"]
    C --> D["驱动在SYSTEM权限<br/>Ring 0上下文运行"]
    D --> E["可访问所有进程内存<br/>截获登录凭证"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style E fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **驱动开发**：编写内核模式驱动，实现 `DriverEntry` 入口点并注册为认证相关组件
2. **注册配置**：通过注册表将驱动注册为 LSASS 加载的安全包或认证提供器
3. **LSASS 加载**：系统启动时 LSASS 进程以 SYSTEM 权限初始化，加载注册的驱动组件
4. **凭证访问**：驱动在 Ring 0 运行，可直接读取 LSASS 进程内存中的所有登录凭证（NTLM哈希、Kerberos票据）

### 主机层检测

**Windows事件ID**：

- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash

```

**用人话说：** LSASS驱动是加载到LSASS（本地安全机构子系统服务）进程中的驱动程序。LSASS是Windows安全策略的核心进程，负责处理认证、密码更改和访问令牌生成。攻击者在`HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services`中注册恶意驱动，使其在LSASS启动时自动加载。因为LSASS以SYSTEM权限运行，恶意驱动获得同等级别的权限，可以访问所有认证相关的数据和操作。
