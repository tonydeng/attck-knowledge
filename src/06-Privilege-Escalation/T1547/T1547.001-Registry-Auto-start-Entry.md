## 这是什么？

**注册表自启项**（T1547.001）是 **启动或登录自动启动执行**（T1547）的一个具体变体，属于 **权限提升** 阶段的攻击技术。

简单来说，就像在电脑启动时自动运行的定时炸弹。

### 具体怎么理解？

在注册表 Run/RunOnce 中添加启动项，最经典的自动启动方法

攻击者使用这种技术时，通常是在权限提升阶段，想要达到特定的攻击目标。与父技术 T1547 相比，T1547.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **权限提升** 阶段使用 注册表自启项 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["攻击者获取注册表<br/>写入权限"] --> B["修改HKCU\Software\Microsoft\<br/>Windows\CurrentVersion\Run"]
    B --> C["添加恶意可执行文件<br/>路径到注册表键值"]
    C --> D["用户下次登录时<br/>Explorer.exe自动加载"]
    D --> E["恶意代码以用户权限<br/>在后台静默执行"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style E fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **注册表写入**：`reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "Updater" /t REG_SZ /d "C:\malware.exe"`
2. **路径选择**：`HKCU` 仅当前用户生效无需管理员权限；`HKLM` 影响所有用户需管理员权限
3. **自动执行**：用户登录时 Explorer.exe 扫描 Run 键下的所有条目并逐一执行
4. **隐蔽变体**：使用 `RunOnce`（执行一次后自动删除）或添加空格伪装参数（如 `C:\legit.exe C:\malware.exe`）

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

**用人话说：** 注册表Run键是Windows中最经典的自动启动机制。攻击者在`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`或`HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`中创建键值，指向恶意程序路径。每次用户登录时Windows自动运行这些程序。例如添加一个名为"WindowsUpdateCheck"的键值指向C:\Users\Public\svchost.exe。使用`reg add`命令或PowerShell的`Set-ItemProperty`即可完成。
