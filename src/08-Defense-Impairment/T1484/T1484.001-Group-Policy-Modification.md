## 这是什么？

**组策略修改**（T1484.001）是 **域或租户策略修改**（T1484）的一个具体变体，属于 **防御削弱** 阶段的攻击技术。

> 📚 **打个比方**：就像攻击者入侵了大楼的管理中心，把安保制度从"严格核查身份"改成了"任何人都可以自由进出"——组策略修改就是攻击者修改Active Directory组策略（GPO），禁用密码策略、关闭防火墙规则、添加特权账户，一次性瘫痪整个域的安全配置。

### 具体怎么理解？

修改Active Directory组策略

攻击者使用这种技术时，通常是在防御削弱阶段，想要达到特定的攻击目标。与父技术 T1484 相比，T1484.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **防御削弱** 阶段使用 组策略修改 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["获取域管理员权限"] --> B["打开组策略管理控制台"]
    B --> C["修改默认域策略GPO"]
    C --> D["禁用密码复杂性要求"]
    D --> E["关闭Windows Defender策略"]
    E --> F["整个域的安全配置被削弱"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **获取域管理员权限** - 使用窃取的域管凭据登录DC
2. **打开GPMC** - 启动组策略管理控制台
3. **修改默认域策略** - 编辑应用于整个域的GPO
4. **禁用安全策略** - 关闭密码策略、审计策略、防火墙规则
5. **整个域安全被削弱** - 所有域成员计算机的安全配置被降低

### 检测思路

检测 组策略修改 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输

```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 攻击者修改Active Directory中的组策略对象（GPO）来削弱整个域的安全配置。比如把"密码复杂度要求"关了、关闭Windows Defender、开放防火墙端口。一个真实案例：攻击者通过修改默认域策略在所有域机器上禁用Windows更新执行恶意脚本。GPO修改的影响范围是整个域或OU，相当于一个人改了整栋大楼的安保规则。如果发现GPO被修改了安全相关设置且修改者不是域管理员团队，这就是域级别的安全事件。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
