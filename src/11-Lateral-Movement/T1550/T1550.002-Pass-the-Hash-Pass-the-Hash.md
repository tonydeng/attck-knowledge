## 这是什么？

**哈希传递**（T1550.002）是 **使用替代认证材料**（T1550）的一个具体变体，属于 **横向移动** 阶段的攻击技术。



> 🔐 **打个比方**：就像破解了门禁系统的密码算法——攻击者破解Kerberos加密来伪造合法的身份凭证。

### 具体怎么理解？

用密码的哈希值代替密码，直接登录远程Windows系统

攻击者使用这种技术时，通常是在横向移动阶段，想要达到特定的攻击目标。与父技术 T1550 相比，T1550.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["攻击者已控制域内主机"] --> B["使用Mimikatz提取NTLM哈希"]
    B --> C["获取管理员账户的密码哈希"]
    C --> D["使用哈希进行NTLM认证"]
    D --> E["通过SMB传递到目标主机"]
    E --> F["执行远程命令控制目标"]
    style E fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **哈希提取**：使用 Mimikatz 的 `sekurlsa::msv` 或 `lsadump::sam` 从 LSASS 内存中提取 NTLM 密码哈希
2. **目标定位**：选择已知的管理员账户哈希，优先选择 Domain Admin 的 NTLM 哈希值
3. **认证传递**：使用 CrackMapExec 或 Mimikatz 的 `sekurlsa::pth` 将哈希值直接用于 NTLM 认证
4. **SMB 连接**：通过 SMB 协议连接到目标主机的 ADMIN$ 或 C$ 管理共享
5. **远程执行**：利用 PsExec 或 WMI 在目标主机上执行命令，获得远程控制权限

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：**

> **检测解读**：监控与哈希传递相关的异常行为，重点关注Windows事件ID 4688, 4104。正常情况下，这些操作应限于IT管理员在工作时间内的合法行为。如果发现非授权用户或异常时间/频率的活动，很可能表明攻击者正在利用哈希传递进行攻击。
>
> **避坑指南**：忽略SMB管理共享（ADMIN$）的异常访问；未启用PowerShell脚本块日志记录；忽略异常的Kerberos票据请求模式。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施
