## 技术描述

**通俗解释：**
Remote Desktop Gateway是远程服务的一个具体实现方式。利用RD网关服务器穿透网络边界，访问内部RDP资源

**技术原理：**
Remote Desktop Gateway（T1021.008）是MITRE ATT&CK框架中定义的一种具体攻击技术。利用RD网关服务器穿透网络边界，访问内部RDP资源攻击者根据具体目标和环境，选择合适的方法和工具来实施该技术。

**用途与影响：**
Remote Desktop Gateway作为远程服务的子技术，在攻击链中扮演着重要角色。成功实施该技术可以帮助攻击者利用RD网关服务器穿透网络边界，访问内部RDP资源，为后续的入侵活动提供支持。

### 应用层检测

**检测方法：** 在应用层监控与Remote Desktop Gateway相关的异常API调用和应用程序行为

**用人话说：**

> RD网关（Remote Desktop Gateway）是微软的RDP代理服务，允许外部用户通过HTTPS隧道访问内网的RDP资源，而不需要直接暴露3389端口到公网。攻击者利用RD网关穿透网络边界——他们拿到一组凭据后，通过RD网关（通常在公司域名的/corp-rpc或/rdweb路径）发起RDP连接，流量走443端口，看起来像HTTPS流量，边界防火墙完全放行。更危险的是，攻击者如果攻陷了RD网关服务器本身，可以监控所有通过网关的RDP流量，窃取会话凭据。检测方法：关注非IT部门的用户账号通过RD网关登录服务器的行为、大量来自不同源IP的RDP连接通过同一网关、以及RD网关日志中的异常登录时间。
>
> **避坑指南**：只监控外部RDP，忽略内网横向RDP。
