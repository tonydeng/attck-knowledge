# Email Account (T1087.003)

## 一句话通俗理解

> **查看邮件系统中的账户**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者入侵后枚举邮件系统中的所有邮箱账户，找出高管和IT管理员 |
| 为什么危险？ | 攻击者通过枚举邮箱，可以精准定位高价值目标发起鱼叉式钓鱼，窃取敏感邮件和凭证 |
| 谁需要关心？ | 邮件管理员、SOC分析师、任何使用Exchange/O365的企业安全人员 |
| 你的第一步防御 | 监控异常的Exchange全局地址列表查询和邮件组成员枚举行为 |
| 如果只做一件事 | 对非IT人员电脑上出现的大量邮箱枚举查询立即告警，因为正常用户不需要遍历整个组织的邮箱列表 |

## 难度等级

⭐ 初级 - 入门级技术，基础操作

## 这是什么？

**电子邮件账户**（T1087.003）是 **账户发现**（T1087）的一个具体变体，属于 **发现** 阶段的攻击技术。



> 📱 **打个比方**：就像查看手机通讯录，看看有哪些联系人——攻击者查看邮箱账号列表，找到高价值目标。

### 具体怎么理解？

查看邮件系统中的账户

攻击者使用这种技术时，通常是在发现阶段，想要达到特定的攻击目标。与父技术 T1087 相比，T1087.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["已入侵内部邮箱或获取OWA访问权"] --> B["枚举Exchange全局地址列表"]
    B --> C["查询邮件组和通讯组成员"]
    C --> D["识别IT管理员和高管邮箱"]
    D --> E["分析邮件组层级关系"]
    E --> F["选定鱼叉式钓鱼的高价值目标"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **地址簿访问**：通过 OWA/EWS 接口或 PowerShell 的 `Get-GlobalAddressList` 枚举邮件账户
2. **组成员查询**：使用 `Get-DistributionGroupMember` 获取各邮件组的成员列表
3. **角色识别**：从账户描述和邮件组名称中识别 IT 管理员、财务、高管等高价值角色
4. **层级分析**：分析邮件组的嵌套关系，理解组织的邮件通信结构
5. **目标选定**：选择最适合作为鱼叉式钓鱼目标的邮箱账户，用于后续凭证窃取

```bash
# 防火墙规则示例：只允许必要的出站连接
```
