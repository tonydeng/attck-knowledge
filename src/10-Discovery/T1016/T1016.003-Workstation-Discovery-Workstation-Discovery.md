# Workstation Discovery (T1016.003)

## 一句话通俗理解

> **发现网络中的工作站和其他计算设备**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者扫描内网，找出网络上哪些电脑在线、是什么系统、属于哪个部门 |
| 为什么危险？ | 攻击者通过扫描找到高价值目标（财务、IT管理电脑），为横向移动选定跳板 |
| 谁需要关心？ | 网络管理员、SOC分析师、安全运维人员 |

| 如果只做一件事 | 对非IT人员电脑上出现的网络扫描命令立即告警，因为正常用户不需要扫描内网 |

## 难度等级

⭐ 初级 - 入门级技术，基础操作

## 这是什么？

**工作站发现**（T1016.003）是 **系统网络配置发现**（T1016）的一个具体变体，属于 **发现** 阶段的攻击技术。



> 💼 **打个比方**：就像看看公司员工的工作站分布图——攻击者查看工作站的归属和用途，寻找有价值的目标机器。

### 具体怎么理解？

发现网络中的工作站和其他计算设备

攻击者使用这种技术时，通常是在发现阶段，想要达到特定的攻击目标。与父技术 T1016 相比，T1016.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["攻击者获取内网访问权"] --> B["扫描内网C段发现活跃主机"]
    B --> C["识别主机操作系统类型"]
    C --> D["查询DNS/NetBIOS获取设备名"]
    D --> E["分析设备角色和用途"]
    E --> F["选定高价值工作站作为横向目标"]
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **子网扫描**：使用 `nmap -sn` 或 `arp -a` 扫描当前所在网段，发现活跃主机
2. **OS 指纹识别**：通过 TTL 值、端口开放情况或 Nmap OS 检测判断目标操作系统类型
3. **设备名查询**：利用 NetBIOS（`nbtstat -A`）或 DNS 反向解析获取主机名称和归属信息
4. **角色分析**：根据设备名（如 `HR-PC01`、`FIN-SERVER`）推断主机用途和所属部门
5. **目标选择**：优先选择财务、IT 管理等高价值工作站作为横向移动目标

检测 工作站发现 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输

```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 这条规则在监控Windows系统中是否有人在执行网络扫描命令。正常情况下，只有网络管理员才会用nmap或arp扫描内网。如果非IT人员的电脑上出现大量内网IP扫描，说明攻击者可能在侦察网络环境，寻找高价值目标。简单说，谁家普通员工会没事扫描公司内网？

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
