# Internet Connection Discovery (T1016.001)

## 一句话通俗理解

> **Internet Connection Discovery就是测试系统能不能连上互联网。**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者入侵后先测试目标系统能不能上网，确认是否有出站连接能力 |
| 为什么危险？ | 这是攻击者建立C2通信通道的前提，能联网意味着攻击者可以远程控制和窃取数据 |
| 谁需要关心？ | 网络管理员、SOC分析师、任何需要检测早期入侵迹象的安全人员 |
| 你的第一步防御 | 监控非IT人员电脑上突然出现的ping、curl、nslookup等网络测试命令 |
| 如果只做一件事 | 对内网主机向外部IP的ping测试建立告警基线，因为正常办公用户不会频繁ping公网 |

## 难度等级

⭐ 初级 - 需要一定的技术基础和实践经验

## 这是什么？

**通俗解释：**
测试系统能不能连上互联网



> 🗺️ **打个比方**：就像入侵者拿到大楼的网络布线图——查看网络配置，知道内网的拓扑结构和出口位置。

**技术原理：**
T1016.001 是 系统网络配置发现（T1016）的子技术，专注于Internet Connection Discovery这一特定方面。攻击者在发现阶段，通过Internet Connection Discovery来获取目标系统或组织的相关信息，为后续攻击步骤做准备。

**用途与影响：**
- 为后续攻击提供关键信息支撑
- 提高攻击的成功率和精准度
- 降低攻击被发现的概率

## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["恶意载荷执行网络检测"] --> B["执行ping 8.8.8.8测试连通性"]
    B --> C["检查系统代理配置"]
    C --> D["测试DNS解析能力"]
    D --> E{"是否可访问互联网?"}
    E -->|"可联网"| F["连接C2服务器接收指令"]
    E -->|"无法联网"| G["切换为离线攻击模式"]
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
    style E fill:#ffa500,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **连通性测试**：执行 `ping` 或 `curl` 测试是否能到达公网 IP，判断系统是否联网
2. **代理检测**：查询注册表或环境变量中的代理配置，分析网络出入口路径
3. **DNS 测试**：尝试解析外部域名，验证 DNS 服务是否可用且未被劫持
4. **环境判定**：综合判断系统是否在隔离环境中（如气隙网络），决定后续通信策略
5. **C2 连接**：确认可联网后，尝试连接预配置的 C2 服务器建立命令控制通道

## 真实案例

1. **确定目标** - 明确要攻击的组织或个人
2. **选择Internet Connection Discovery方法** - 根据目标特点选择最有效的收集方法
3. **执行Internet Connection Discovery操作** - 使用相应的工具和技术进行信息收集
4. **整理和分析信息** - 对收集到的数据进行分类和验证
5. **利用信息进行下一步攻击** - 基于收集到的信息制定后续攻击计划

- **时间**: 2023-2024年

### 实战技巧

1. **隐蔽优先**：在发现阶段使用被动方式收集信息，避免触发安全告警
2. **信息验证**：对收集到的信息进行交叉验证，确保准确性和时效性
3. **工具选择**：根据目标环境选择合适的工具，避免使用已被广泛检测的工具
4. **OPSEC意识**：使用匿名网络、临时环境进行操作，防止溯源

- 记录操作日志，用于后续分析和报告编写

## 蓝队视角

### 检测要点

1. **异常信息收集行为**：监控来自内部系统的异常数据查询和收集行为
2. **可疑工具使用**：检测与Internet Connection Discovery相关的工具在内部网络中的使用
3. **异常网络流量**：监控对外部信息收集平台的可疑网络连接
4. **权限异常**：关注非授权用户的信息收集和查询行为

### 监控建议

- 部署信息收集行为的检测规则
- 建立基准行为模型，及时发现异常
- 定期审计敏感信息的访问记录

## 检测建议

### 网络层检测

**检测方法：** 监控与Internet Connection Discovery相关的网络流量特征

**具体规则/命令示例：**

```bash
# 监控异常DNS查询
tcpdump -i eth0 port 53 | grep -E "可疑域名"
```

### 主机层检测

**Windows事件ID：**

- 事件ID 4688：可疑进程创建
- 事件ID 4104：PowerShell脚本块日志

**Linux日志：**

- 日志文件：`/var/log/syslog`
- 关键字段：可疑命令执行

### 应用层检测

**用人话说：** 这条Sigma规则在监控Windows系统中是否有人在执行网络连通性测试命令。正常情况下，只有IT人员才会频繁ping公网IP地址。如果非IT人员的电脑上出现大量ping 8.8.8.8或curl外部URL的记录，说明有人可能在测试网络出口，为后续C2通信做准备。简单说，谁家办公人员会没事ping谷歌DNS？

**Sigma规则示例：**

```yaml

  - attack.T1016/001
```

## 缓解措施

### 优先级1：关键措施

**措施名称：** 敏感信息保护

**具体实施步骤：**

1. 识别和分类组织内的敏感信息
2. 对敏感信息实施访问控制和加密
3. 部署信息泄露防护（DLP）解决方案

### 优先级2：重要措施

**措施名称：** 员工安全意识培训

**具体实施步骤：**

1. 定期开展信息安全意识培训
2. 教育员工识别社交工程攻击
3. 建立信息报告和响应机制

### 优先级3：建议措施

**措施名称：** 安全配置加固

**具体实施步骤：**

1. 限制公开可访问的系统信息
2. 配置合适的日志记录和告警策略
3. 定期进行安全评估和渗透测试
