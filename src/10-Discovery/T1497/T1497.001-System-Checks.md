# 系统检查 (T1497.001)

## 一句话通俗理解

> **检查CPU、BIOS、MAC地址等硬件特征判断是否在虚拟机中**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 恶意软件通过检查硬件特征来判断自己是否在虚拟机或沙箱中运行 |
| 为什么危险？ | 如果检测到在分析环境中，恶意软件会终止执行或释放诱饵载荷，让安全分析人员看不到真实攻击行为 |

## 难度等级

⭐⭐⭐ 高级 - 需要深入的技术知识和实践

## 这是什么？

**系统检查**（T1497.001）是 **虚拟化/沙箱规避**（T1497）的一个具体变体，属于 **发现** 阶段的攻击技术。



> 🕵️ **打个比方**：就像小偷进屋前先试探一下有没有报警器——攻击者检查系统环境（进程、驱动、注册表），确认是否在沙箱或虚拟机中。

### 具体怎么理解？

检查CPU、BIOS、MAC地址等硬件特征判断是否在虚拟机中

攻击者使用这种技术时，通常是在发现阶段，想要达到特定的攻击目标。与父技术 T1497 相比，T1497.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["恶意载荷在目标系统启动"] --> B["查询CPU核心数和型号"]
    B --> C["检查BIOS信息和制造商"]
    C --> D["枚举虚拟机驱动(vmtools/vboxguest)"]
    D --> E["比对MAC地址前缀是否为VM厂商"]
    E --> F{"是否发现虚拟化特征?"}
    F -->|"检测到VM/沙箱特征"| G["判定为分析环境,终止执行"]
    F -->|"未发现异常特征"| H["确认真实环境,继续攻击"]
    style G fill:#ff6b6b,stroke:#333,stroke-width:2px
    style F fill:#ffa500,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **硬件指纹采集**：通过 `cpuid` 指令或 WMI 查询获取 CPU 核心数、型号信息，沙箱通常配置较少的核心数
2. **BIOS 信息提取**：读取注册表或 SMBIOS 数据，检查 BIOS 制造商是否为 VMware、VirtualBox 等虚拟化厂商
3. **驱动枚举**：列举系统中加载的设备驱动，查找 `vmtools.sys`、`VBoxGuest.sys` 等虚拟化工具驱动
4. **MAC 地址比对**：获取网卡 MAC 地址前缀，与已知虚拟化厂商 OUI（如 00:0C:29 为 VMware）进行比对
5. **综合判定**：根据多项指标综合判断是否在虚拟机中运行，决定是否终止恶意行为

## 蓝队视角

### 检测要点

1. **系统日志监控**：关注与系统检查相关的系统日志和审计记录
2. **异常行为检测**：监控系统中与该技术相关的异常进程、网络连接和文件操作
3. **工具特征识别**：识别攻击者可能使用的工具在系统中的运行特征

### 监控建议

- 部署端点检测和响应（EDR）系统，监控与系统检查相关的异常行为
- 配置SIEM规则，关联分析来自多个来源的告警
- 定期进行安全评估和渗透测试，验证检测规则的有效性

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
