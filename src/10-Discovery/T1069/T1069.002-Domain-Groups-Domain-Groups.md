# Domain Groups (T1069.002)

## 一句话通俗理解

> **查看域中的安全组和成员**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 攻击者入侵后枚举整个域环境中的安全组，找出Domain Admins等高权限组的成员 |
| 为什么危险？ | 攻击者通过枚举域组，可以精准定位域管理员账户，为凭证窃取和横向移动提供高价值目标 |
| 谁需要关心？ | 域管理员、SOC分析师、任何维护Active Directory环境的安全人员 |
| 你的第一步防御 | 监控 `net group /domain`、`Get-DomainGroup` 等域组枚举命令的执行 |
| 如果只做一件事 | 对非IT人员电脑上出现的域组枚举命令立即告警，因为正常办公用户不需要查看整个域的安全组列表 |

## 难度等级

⭐⭐ 中级 - 需要一定的技术基础和经验

## 这是什么？

**域组**（T1069.002）是 **权限组发现**（T1069）的一个具体变体，属于 **发现** 阶段的攻击技术。



> 🔑 **打个比方**：就像查看公司的门禁权限登记表——查看域组的成员关系，找到拥有最高权限的账户。

### 具体怎么理解？

查看域中的安全组和成员

攻击者使用这种技术时，通常是在发现阶段，想要达到特定的攻击目标。与父技术 T1069 相比，T1069.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

```mermaid
graph TD
    A["已获取域内合法凭据"] --> B["执行net group /domain枚举域组"]
    B --> C["查询Domain Admins组成员"]
    C --> D["分析Enterprise Admins等高权限组"]
    D --> E["使用BloodHound映射组嵌套关系"]
    E --> F["识别域管理员和关键服务账户"]
    style F fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **组枚举**：使用 `net group /domain` 或 PowerView 的 `Get-DomainGroup` 列出所有域安全组
2. **特权组查询**：重点查询 `Domain Admins`、`Enterprise Admins`、`Schema Admins` 等高权限组的成员
3. **嵌套分析**：分析组的嵌套关系，某些普通组可能通过嵌套获得管理员权限
4. **映射可视化**：使用 BloodHound/SharpHound 生成组关系图，直观展示权限传递路径
5. **目标定位**：确定域管理员账户和关键服务账户，为后续凭证窃取和横向移动提供目标

检测 域组 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输
# 根据具体协议和端口设置检测规则
```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 这条规则在监控Windows系统中是否有人在枚举域安全组。正常情况下，只有域管理员才会用 `net group /domain` 或 `Get-DomainGroup` 查看整个域的安全组列表。如果非IT人员的电脑上出现这类命令，说明攻击者可能在侦察域权限结构，寻找Domain Admins等高权限组的成员。简单说，谁家普通员工会没事查看整个公司的安全组和管理员名单？

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
