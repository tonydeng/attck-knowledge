## 这是什么？

**虚拟专用服务器(VPS)**（T1584.003）是 **破坏基础设施**（T1584）的一个具体变体，属于 **资源开发** 阶段的攻击技术。



> 📶 **打个比方**：就像蹭别人家的WiFi发匿名邮件——攻陷第三方基础设施让攻击行为更难溯源。

### 具体怎么理解？

入侵别人的VPS当跳板

攻击者使用这种技术时，通常是在资源开发阶段，想要达到特定的攻击目标。与父技术 T1584 相比，T1584.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程


```mermaid
graph TD
    A["扫描互联网上的脆弱VPS实例"]
    B["利用SSH弱密码或已知漏洞入侵"]
    C["安装后门和持久化工具"]
    D["利用被入侵VPS作为跳板机"]
    E["隐藏真实身份发起攻击"]
    A --> B
    B --> C
    C --> D
    D --> E
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **扫描脆弱VPS** - 攻击者扫描互联网上存在SSH弱密码、未修补漏洞或默认配置的VPS实例
2. **入侵VPS系统** - 通过暴力破解SSH密码、利用已知漏洞或Web应用漏洞获得VPS的root权限
3. **植入后门** - 安装Rootkit、后门程序和持久化机制，确保长期控制权，这是控制基础设施的关键
4. **配置跳板** - 将入侵的VPS配置为代理跳板或中继节点，用于转发攻击流量
5. **隐藏真实身份** - 利用被入侵的VPS发起后续攻击，溯源时只会追踪到被入侵的VPS所有者

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
