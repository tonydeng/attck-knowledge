## 这是什么？

**数字证书**（T1596.003）是 **搜索开放技术数据库**（T1596）的一个具体变体，属于 **侦察** 阶段的攻击技术。



> 📋 **打个比方**：就像通过公开的企业黄页找线索——攻击者从数字证书透明度日志中查找目标的所有SSL/TLS证书。

### 具体怎么理解？

查询SSL/TLS证书信息，发现子域名和服务

攻击者使用这种技术时，通常是在侦察阶段，想要达到特定的攻击目标。与父技术 T1596 相比，T1596.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["确定目标域名"] --> B["查询证书透明度日志获取所有SSL证书"]
    B --> C["从证书SAN字段提取所有子域名"]
    C --> D["分析证书颁发机构和使用类型"]
    D --> E["发现未公开的子域名和测试环境"]
    style B fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **域名确定** - 确定要查询证书信息的目标域名
2. **CT日志查询** - 使用crt.sh或Censys查询证书透明度日志
3. **SAN提取** - 从证书的主题备用名称（SAN）字段提取所有子域名
4. **证书分析** - 分析证书类型（DV/OV/EV）、颁发机构和有效期
5. **资产发现** - 发现未在DNS中公开的测试子域名和内部域名

```bash

```

### 应用层检测


**用人话说：** 攻击者像翻看目标公司的通信记录一样，利用证书透明度（Certificate Transparency）日志——比如crt.sh——查询目标域名下签发过的所有SSL/TLS证书。每张SSL证书都会记录域名和子域名信息，而证书透明度日志是公开的。攻击者查crt.sh输入"company.com"，返回的列表里可能包含"mail.company.com、vpn.company.com、jenkins.company.com、gitlab.internal.company.com"等大量子域名——有些可能是开发团队忘了隐藏的、直接暴露在公网上的内部系统。黑客不需要扫描你们网站，只需要查一下证书日志就能拿到你们的域名资产清单。如果你看到有人在短时间内通过crt.sh等查询服务搜索你们公司的域名，这就是在绘制完整的子域名攻击面。
