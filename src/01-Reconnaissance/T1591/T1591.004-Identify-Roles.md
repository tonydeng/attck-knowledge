## 这是什么？

**识别角色**（T1591.004）是 **收集受害者组织信息**（T1591）的一个具体变体，属于 **侦察** 阶段的攻击技术。



> 🎭 **打个比方**：就像骗子先搞清楚公司里谁是财务、谁是IT、谁是高管——知道谁管钱、谁管系统，才能精准冒充。

### 具体怎么理解？

了解目标公司的组织架构，谁是关键人物

攻击者使用这种技术时，通常是在侦察阶段，想要达到特定的攻击目标。与父技术 T1591 相比，T1591.004 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid

    A["在LinkedIn上搜索目标公司员工信息"] --> B["识别关键岗位人员——IT运维、财务、高管"]

    C --> D["确定拥有高权限访问的员工账户"]
    D --> E["将高价值目标列入凭证收集优先清单"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **员工搜索** - 在LinkedIn、猎聘等平台搜索目标公司的在职员工
2. **角色识别** - 找出IT管理员、系统运维、财务主管和高管等关键角色
3. **职责分析** - 查看员工的工作描述了解其系统访问权限和技术背景
4. **权限评估** - 根据岗位判断可能拥有的系统管理和数据访问权限级别
5. **优先排序** - 将高权限员工列入社会工程学和凭证收集的优先目标

### 应用层检测


**用人话说：** 攻击者像弄清楚谁是保安队长、谁是金库管理员一样，识别目标公司的关键人物：IT运维、系统管理员、财务总监、CEO助理等。他们在LinkedIn上搜索目标公司的员工，通过职位描述判断谁有系统管理权限、谁管钱、谁接触核心数据。比如找到一个人的头衔是"Senior Network Administrator"，就知道他可能有防火墙和路由器的管理权限；找到"VP of Finance"，就知道他是BEC（商务邮件欺诈）的高价值目标。有了这些信息，攻击者就能制定精准的社会工程学方案——冒充IT管理员给财务发邮件，或者冒充CEO给IT要求重置密码。正常HR或猎头也会看员工资料，但如果有人专门针对某些权限岗位进行系统性梳理和分类，这就是攻击前的目标筛选。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
