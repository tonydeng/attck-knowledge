## 这是什么？

**识别业务关系**（T1591.002）是 **收集受害者组织信息**（T1591）的一个具体变体，属于 **侦察** 阶段的攻击技术。



> 🤝 **打个比方**：就像骗子先研究你和谁做生意、有什么合作伙伴——知道业务关系，才能伪装身份接近你。

### 具体怎么理解？

了解目标公司跟哪些供应商、合作伙伴有业务往来

攻击者使用这种技术时，通常是在侦察阶段，想要达到特定的攻击目标。与父技术 T1591 相比，T1591.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["搜索目标公司的新闻稿和合作伙伴公告"] --> B["分析官网展示的客户案例和供应商名单"]

    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **公开信息收集** - 搜索目标公司的新闻稿、年报和合作伙伴公告
2. **案例分析** - 分析官网展示的成功案例、客户评价和供应商列表
3. **关系识别** - 识别关键上游供应商和下游客户之间的依赖关系
4. **网络绘制** - 绘制完整的供应链和业务关系网络图
5. **弱点评估** - 评估各业务伙伴的安全水平，选择薄弱环节作为突破口

### 应用层检测


**用人话说：** 攻击者像绘制社交关系图一样，摸清目标公司的上下游合作伙伴、客户、供应商和技术外包商。他们从新闻稿中的"战略合作"公告、官网展示的客户案例、供应商名录、甚至LinkedIn上两家公司员工的互关关系中梳理出业务关系网。比如发现目标公司的人力资源系统由某家SaaS厂商提供，攻击者就会去研究这家SaaS厂商的安全性——如果这家厂商有漏洞，攻破它就能间接获取目标公司的员工数据。正常市场调研也会分析业务关系，但如果有人专门针对目标公司的供应商进行安全评估和信息收集，且目标公司的合作伙伴信息被系统化地归档整理，这就是供应链攻击的前期准备。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
