## 这是什么？

**搜索引擎**（T1593.002）是 **搜索开放网站/域名**（T1593）的一个具体变体，属于 **侦察** 阶段的攻击技术。



> 🌐 **打个比方**：就像在百度上搜自己的名字，发现原来那么多个人信息都挂在网上——搜索引擎就是攻击者的"信息金矿"。

### 具体怎么理解？

使用Google等搜索引擎的高级语法搜索敏感信息

攻击者使用这种技术时，通常是在侦察阶段，想要达到特定的攻击目标。与父技术 T1593 相比，T1593.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用

```mermaid

    A["使用Google Dorking构造高级搜索语法"] --> B["搜索目标域下暴露的文档和配置文件"]

```

**步骤详解：**

1. **Dork构造** - 使用site:filetype:等Google高级搜索语法定位目标信息
2. **文档搜索** - 搜索暴露的PDF、Excel、Word文档中可能包含的敏感信息
3. **凭证查找** - 使用password、passwd、key等关键词搜索泄露的凭证
4. **目录发现** - 查找Web目录列表、备份文件和未授权的管理页面
5. **信息汇总** - 将搜索引擎收集的所有敏感信息分类整理


### 应用层检测


**用人话说：** 攻击者像用高级搜法在互联网上挖宝一样，使用Google Dorking等搜索引擎高级语法搜索目标公司意外暴露的敏感信息。比如用"site:company.com filetype:xls password"搜索公司域名下含有密码的Excel文件；用"site:company.com intitle:index of"搜索开放了目录列表的Web服务器。这些搜索能发现：暴露的配置文件（.env、config.php）、包含数据库连接串的备份文件、未授权访问的管理后台、甚至包含员工密码的旧版本通讯录。攻击者坐在电脑前，不接触目标网络一分一毫，就能通过搜索引擎找到大量敏感信息。如果你在搜索引擎中搜自己公司的域名发现了很多意料之外的文件，说明你们的信息暴露面比想象中大得多。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
