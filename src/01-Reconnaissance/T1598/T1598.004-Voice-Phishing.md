> **通过电话冒充可信身份，骗取目标的敏感信息**

## 这是什么？

**语音型钓鱼**（T1598.004）是 **钓鱼获取信息**（T1598）的一个具体变体，属于 **侦察** 阶段的攻击技术。



> 📱 **打个比方**：就像冒充客服发短信说你的账号有风险——攻击者通过SMS钓鱼（Smishing）发送恶意链接或诱骗回复敏感信息。

### 具体怎么理解？

通过电话冒充可信身份，骗取目标的敏感信息

攻击者使用这种技术时，通常是在侦察阶段，想要达到特定的攻击目标。与父技术 T1598 相比，T1598.004 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["收集目标员工信息用于冒充对象"] --> B["伪造来电显示为IT部门或银行客服"]
    B --> C["编造紧急事由制造心理压力"]
    C --> D["诱导目标提供系统凭据或验证码"]
    D --> E["利用获取的信息立即进行账户接管"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **信息收集** - 收集目标员工的姓名、职位和所属部门等信息用于冒充
2. **号码伪造** - 使用VoIP服务伪造来电显示为可信机构号码
3. **紧急话术** - 编造系统故障、账户异常等紧急事由制造心理压力
4. **凭证骗取** - 以验证身份为理由要求目标提供密码或短信验证码
5. **立即利用** - 获取凭证后迅速登录目标账户完成信息收集

### 应用层检测


**用人话说：** 攻击者像冒充银行客服打电话要你的验证码一样，通过电话（Vishing）或语音邮件（Voicemail Phishing）直接与目标对话骗取敏感信息。攻击者会冒充IT支持人员打电话给财务："我是IT部门的张三，系统升级后你的邮箱密码需要重置，请告诉我你的新密码"；或者冒充银行风控："我们发现你账户有一笔可疑交易，请提供你的短信验证码以确认身份"。语音钓鱼的可怕之处在于：人类声音天然带有信任感，而语言交流又让人来不及仔细思考。有些攻击者还会用AI语音克隆工具模仿高管的口音给手下打电话要求紧急转账。如果你发现有人冒充内部IT、安全团队或业务合作方的身份打电话给员工索要密码、验证码或凭证，这就是语音钓鱼攻击。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
