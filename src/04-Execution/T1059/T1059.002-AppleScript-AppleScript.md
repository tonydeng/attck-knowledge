> **AppleScript就是macOS的自动化脚本语言。**

## 这是什么？

**通俗解释：**
macOS的自动化脚本语言，可以控制应用程序和系统操作



> 🍎 **打个比方**：就像用iPhone上的快捷指令自动完成任务——AppleScript是macOS上的自动化脚本工具，攻击者用来执行恶意操作。

**技术原理：**
T1059.002 是 命令和脚本解释器（T1059）的子技术，专注于AppleScript这一特定方面。攻击者在执行阶段，通过AppleScript来获取目标系统或组织的相关信息，为后续攻击步骤做准备。

**用途与影响：**
- 为后续攻击提供关键信息支撑
- 提高攻击的成功率和精准度
- 降低攻击被发现的概率

## 真实攻击流程

```mermaid
graph TD
    A["诱导用户运行AppleScript"] --> B["通过脚本访问系统权限"]
    B --> C["控制其他macOS应用程序"]
    C --> D["窃取敏感数据或执行命令"]
    D --> E["持久化到Login Items启动项"]
    style B fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **诱导用户运行AppleScript** - 将恶意脚本伪装成.app捆绑包或.scpt文件通过钓鱼邮件投递
2. **通过脚本访问系统权限** - 利用osascript命令请求辅助功能或完全磁盘访问权限
3. **控制其他macOS应用程序** - 通过tell application语法控制邮件、浏览器等应用执行恶意操作
4. **窃取敏感数据或执行命令** - 使用do shell script执行任意Shell命令，窃取文件或密码
5. **持久化到Login Items启动项** - 将脚本添加到登录项中，实现系统启动时自动运行

## 真实案例

### 案例1：APT组织使用AppleScript进行攻击准备

- **时间**: 2023-2024年
- **目标**: 多行业目标组织
- **攻击组织**: 多个APT组织
- **手法**: 攻击者在攻击初期大量使用AppleScript技术收集目标信息，为后续定向攻击做准备
- **影响**: 攻击成功率显著提高，防御者难以及时发现侦察行为
- **参考链接**: [MITRE ATT&CK - T1059.002](https://attack.mitre.org/techniques/T1059/002/)

### 案例2：红队演练中的AppleScript应用

- **时间**: 2024-2025年
- **目标**: 授权测试的企业客户
- **攻击组织**: 红队团队
- **手法**: 在授权的红队演练中，AppleScript被用于模拟真实攻击者的信息收集行为，测试企业安全监控体系能否及时发现侦察活动
- **影响**: 帮助企业发现信息暴露面和安全监控盲区
- **参考链接**: 红队演练报告（内部资料）

### 实战技巧

1. **隐蔽优先**：在执行阶段使用被动方式收集信息，避免触发安全告警
2. **信息验证**：对收集到的信息进行交叉验证，确保准确性和时效性
3. **工具选择**：根据目标环境选择合适的工具，避免使用已被广泛检测的工具
4. **OPSEC意识**：使用匿名网络、临时环境进行操作，防止溯源

## 蓝队视角

### 检测要点

1. **异常信息收集行为**：监控来自内部系统的异常数据查询和收集行为
2. **可疑工具使用**：检测与AppleScript相关的工具在内部网络中的使用
3. **异常网络流量**：监控对外部信息收集平台的可疑网络连接
4. **权限异常**：关注非授权用户的信息收集和查询行为

### 监控建议

- 部署信息收集行为的检测规则
- 建立基准行为模型，及时发现异常
- 定期审计敏感信息的访问记录

## 检测建议

### 网络层检测

**检测方法：** 监控与AppleScript相关的网络流量特征

**具体规则/命令示例：**

```bash
# 监控异常DNS查询
tcpdump -i eth0 port 53 | grep -E "可疑域名"
```

### 主机层检测

**Windows事件ID：**

- 事件ID 4688：可疑进程创建
- 事件ID 4104：PowerShell脚本块日志

**Linux日志：**

- 日志文件：`/var/log/syslog`
- 关键字段：可疑命令执行

### 应用层检测

**用人话说：** 攻击者用osascript执行AppleScript在macOS上自动化恶意操作——可以控制浏览器、模拟点击、甚至窃取密码。典型恶意用法：`osascript -e 'tell app "Safari" to open URL "http://malicious.com"'`控制浏览器访问钓鱼站，或`osascript -e 'tell app "System Events" to keystroke "password"'`窃取键盘输入。攻击者会将AppleScript藏在.app捆绑包的Scripts.applescript文件中绕过Gatekeeper安全检查。检测上，关注osascript进程的异常父进程（如从Safari/Chrome或不明应用启动）、以及osascript读取了系统钥匙串或浏览器Cookie的行为。

**Sigma规则示例：**

## 缓解措施

### 优先级1：关键措施

**措施名称：** 敏感信息保护

**具体实施步骤：**

1. 识别和分类组织内的敏感信息
2. 对敏感信息实施访问控制和加密
3. 部署信息泄露防护（DLP）解决方案

### 优先级2：重要措施

**措施名称：** 员工安全意识培训

**具体实施步骤：**

1. 定期开展信息安全意识培训
2. 教育员工识别社交工程攻击
3. 建立信息报告和响应机制

### 优先级3：建议措施

**措施名称：** 安全配置加固

**具体实施步骤：**

1. 限制公开可访问的系统信息
2. 配置合适的日志记录和告警策略
3. 定期进行安全评估和渗透测试
