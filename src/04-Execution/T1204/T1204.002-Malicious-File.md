# 恶意文件 (T1204.002)

## 一句话通俗理解

> **骗用户打开恶意附件或下载的文件**

## 30秒速查卡

| 这是什么？ | 通过钓鱼邮件或网站诱导用户打开恶意文件 |
| 为什么危险？ | 攻击者用它投递恶意文档、PDF、程序，用户打开就中招 |
| 谁需要关心？ | 企业员工、邮件安全团队、SOC分析师 |

## 难度等级

⭐ 初级 - 入门级技术，基础操作

## 这是什么？

**恶意文件**（T1204.002）是 **用户执行**（T1204）的一个具体变体，属于 **执行** 阶段的攻击技术。



> 📎 **打个比方**：就像收到一封带附件的邮件，附件里藏了病毒——攻击者诱导用户打开恶意文件（Office文档、PDF等），触发恶意代码执行。

### 具体怎么理解？

骗用户打开恶意附件或下载的文件

攻击者使用这种技术时，通常是在执行阶段，想要达到特定的攻击目标。与父技术 T1204 相比，T1204.002 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["制作包含恶意宏的Office或PDF文件"] --> B["通过邮件附件投递给目标"]
    B --> C["诱导用户打开附件"]
    C --> D["触发宏或漏洞执行恶意代码"]
    D --> E["下载并执行后续Payload"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **制作包含恶意宏的Office或PDF文件** - 在Word/Excel中嵌入VBA宏或利用PDF漏洞制作恶意文件
2. **通过邮件附件投递给目标** - 将恶意文件伪装成发票、简历、订单等诱饵文件通过邮件发送
3. **诱导用户打开附件** - 利用社会工程学话术欺骗用户双击打开附件文件
4. **触发宏或漏洞执行恶意代码** - 用户打开后自动触发宏代码或利用文档解析漏洞执行ShellCode
5. **下载并执行后续Payload** - 宏或漏洞代码下载并执行后续恶意载荷，如Cobalt Strike Beacon

## 检测建议

### 检测思路

检测 恶意文件 的关键是识别异常行为模式。以下是三个层面的检测方法：

### 网络层检测

**方法**：监控网络流量中的异常模式

```bash
# 检测异常的网络连接和数据传输
# 根据具体协议和端口设置检测规则
```

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 用户自己双击了恶意文件——攻击者利用人的好奇心而非系统漏洞。常用伎俩：双扩展名如invoice.pdf.exe（Windows默认隐藏已知扩展名）、自解压RAR（双击自动解压释放恶意软件）、或LNK快捷方式指向远程恶意脚本。经典案例：Emotet通过恶意Word附件传播，用户双击后启用宏→下载Emotet DLL→感染整个内网。检测上，关注由explorer.exe（资源管理器）直接启动的可疑可执行文件，特别是从%TEMP%、Downloads或AppData目录启动的进程文件名包含多个扩展名或看似合法实为乱码（如invoice.pdf.exe、document.pdf.scr）。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```

### 优先级2：重要措施

**加强监控**：部署EDR和SIEM系统，实时监控异常行为

### 优先级3：建议措施

**安全意识培训**：教育员工识别相关攻击手法
