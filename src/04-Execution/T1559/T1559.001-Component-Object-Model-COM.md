## 这是什么？

**COM**（T1559.001）是 **进程间通信**（T1559）的一个具体变体，属于 **执行** 阶段的攻击技术。



> 📧 **打个比方**：就像通过公司内部邮件系统发一条"帮你重启电脑"的指令——攻击者利用COM接口在不同进程间通信和执行代码。

### 具体怎么理解？

利用Windows组件对象模型在合法进程中执行代码

攻击者使用这种技术时，通常是在执行阶段，想要达到特定的攻击目标。与父技术 T1559 相比，T1559.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["注册恶意COM组件"] --> B["通过合法程序调用COM接口"]
    B --> C["利用COM提权漏洞提升权限"]
    C --> D["在目标进程中注入并执行代码"]
    D --> E["横向迁移到高权限系统进程"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **注册恶意COM组件** - 在Windows注册表中注册恶意COM服务器，设置CLSID和AppID
2. **通过合法程序调用COM接口** - 利用Office或浏览器等合法程序通过COM接口与恶意组件通信
3. **利用COM提权漏洞提升权限** - 利用COM的提权漏洞（如EoP）从低权限用户提升到SYSTEM权限
4. **在目标进程中注入并执行代码** - 利用COM组件的进程内激活特性，在合法进程中执行ShellCode
5. **横向迁移到高权限系统进程** - 通过COM跨进程通信将控制转移到更具权限的系统进程中

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 攻击者利用Windows COM组件在合法进程中执行代码——像在别人家客厅搞破坏，别人还以为是主人自己干的。常用技术：通过Office调用MMC20.Application COM对象启动子进程`powershell -e ...`，或注册恶意COM对象劫持系统组件。红队工具Cobalt Strike的execute-assembly通过COM在内存中加载.NET程序集（不落地文件）。检测上，关注Office进程（winword/excel）启动的MMC/PowerShell子进程（事件ID 4697）、以及注册表HKEY_CURRENT_USER\Software\Classes\CLSID下新增的COM对象注册——没有管理员权限的普通应用不应该注册COM组件。

## 缓解措施

### 优先级1：关键措施

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
iptables -A OUTPUT -p tcp --dport 443 -j ACCEPT  # 允许HTTPS
iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT   # 允许HTTP
iptables -A OUTPUT -p tcp -j DROP                 # 阻止其他TCP
```
