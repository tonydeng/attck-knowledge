# DLL搜索顺序劫持 (T1574.001)

## 一句话通俗理解

> **在合法DLL之前放置恶意同名DLL**

## 这是什么？

**DLL搜索顺序劫持**（T1574.001）是 **劫持执行流**（T1574）的一个具体变体，属于 **执行** 阶段的攻击技术。

简单来说，就像在高速公路的出口设个假路标，把车引到错误的方向。

### 具体怎么理解？

在合法DLL之前放置恶意同名DLL

攻击者使用这种技术时，通常是在执行阶段，想要达到特定的攻击目标。与父技术 T1574 相比，T1574.001 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

```mermaid
graph TD
    A["识别目标程序缺失的DLL依赖"] --> B["在程序搜索路径中放置恶意DLL"]
    B --> C["恶意DLL与合法DLL同名"]
    C --> D["程序启动时优先加载恶意DLL"]
    D --> E["DllMain入口自动执行ShellCode"]
    style D fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **识别目标程序缺失的DLL依赖** - 使用Process Monitor分析目标程序启动时找不到的DLL文件
2. **在程序搜索路径中放置恶意DLL** - 将恶意DLL放置在程序当前目录等优先搜索路径中
3. **恶意DLL与合法DLL同名** - 恶意DLL使用与被劫持DLL完全相同的文件名
4. **程序启动时优先加载恶意DLL** - 程序按DLL搜索顺序先找到并加载恶意DLL而非系统目录中的原版
5. **DllMain入口自动执行ShellCode** - DLL加载时DllMain函数自动执行，将恶意代码注入合法进程

### 主机层检测

**Windows事件ID**：

- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测

**用人话说：** 攻击者利用Windows DLL搜索顺序机制——程序在加载DLL时先查当前目录再查系统目录，攻击者就在目标程序目录放同名恶意DLL。比如某软件启动时加载"vcruntime140.dll"但没指定绝对路径，攻击者在同一目录放同名恶意DLL，程序就会加载假的。红队工具Robber可扫描哪些程序存在DLL劫持漏洞。检测上，关注进程启动时从非标准路径加载了系统级DLL（如从%TEMP%或程序安装目录加载ntdll.dll/kernel32.dll），以及使用Process Monitor查看进程加载了不属于系统路径的DLL模块。
