## 这是什么？

**通俗解释：**
Linux的服务管理工具，用于控制systemd服务



> 🗂️ **打个比方**：就像用Windows的任务管理器远程创建一个新任务——攻击者通过Windows Task Scheduler在远程系统上创建和执行任务。

**技术原理：**
T1569.003 是 系统服务（T1569）的子技术，专注于Systemctl这一特定方面。攻击者在执行阶段，通过Systemctl来获取目标系统或组织的相关信息，为后续攻击步骤做准备。

**用途与影响：**
- 为后续攻击提供关键信息支撑
- 提高攻击的成功率和精准度
- 降低攻击被发现的概率

## 真实攻击流程

```mermaid
graph TD
    A["创建恶意systemd service单元文件"] --> B["将service文件写入系统目录"]
    B --> C["通过systemctl enable启用服务"]
    C --> D["设置Restart=always实现自动重启"]
    D --> E["以root权限启动并执行恶意代码"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **创建恶意systemd service单元文件** - 编写包含恶意命令的.service文件，指定ExecStart为恶意程序路径
2. **将service文件写入系统目录** - 将.service文件复制到 `/etc/systemd/system/` 或 `/usr/lib/systemd/system/`
3. **通过systemctl enable启用服务** - 执行 `systemctl enable malicious.service` 创建符号链接启用开机自启
4. **设置Restart=always实现自动重启** - 在service配置中设置Restart=always，退出后自动重启保持驻留
5. **以root权限启动并执行恶意代码** - 执行 `systemctl start malicious.service`，恶意程序以root权限运行

- **时间**: 2023-2024年
- **目标**: 多行业目标组织

- **手法**: 攻击者在攻击初期大量使用Systemctl技术收集目标信息，为后续定向攻击做准备

### 实战技巧

1. **隐蔽优先**：在执行阶段使用被动方式收集信息，避免触发安全告警
2. **信息验证**：对收集到的信息进行交叉验证，确保准确性和时效性
3. **工具选择**：根据目标环境选择合适的工具，避免使用已被广泛检测的工具
4. **OPSEC意识**：使用匿名网络、临时环境进行操作，防止溯源

## 蓝队视角

### 检测要点

1. **异常信息收集行为**：监控来自内部系统的异常数据查询和收集行为
2. **可疑工具使用**：检测与Systemctl相关的工具在内部网络中的使用
3. **异常网络流量**：监控对外部信息收集平台的可疑网络连接
4. **权限异常**：关注非授权用户的信息收集和查询行为

### 监控建议

- 部署信息收集行为的检测规则
- 建立基准行为模型，及时发现异常
- 定期审计敏感信息的访问记录

## 检测建议

### 网络层检测

**检测方法：** 监控与Systemctl相关的网络流量特征

**具体规则/命令示例：**

```bash
# 监控异常DNS查询
tcpdump -i eth0 port 53 | grep -E "可疑域名"
```

### 主机层检测

**Windows事件ID：**

- 事件ID 4688：可疑进程创建
- 事件ID 4104：PowerShell脚本块日志

**Linux日志：**

- 日志文件：`/var/log/syslog`
- 关键字段：可疑命令执行

### 应用层检测

**用人话说：** 攻击者用systemctl在Linux上创建systemd服务实现持久化和执行——Linux生态的标准服务管理。攻击者写.service文件如/etc/systemd/system/systemd-network-update.service，内容含`ExecStart=/bin/bash -c "/tmp/.evil"`，然后`systemctl enable --now systemd-network-update.service`启用。恶意服务常伪装成正常系统名（如ssh-agent、systemd-logind），并设`Restart=always`确保崩溃后复活。检测上，关注/etc/systemd/system/下新创建的.service文件（关注ExecStart指向可疑命令）、以及由非root用户或Web服务进程触发的systemctl enable/systemctl start命令。
