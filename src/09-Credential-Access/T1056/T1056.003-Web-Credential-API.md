# Web凭证API (T1056.003)

## 一句话通俗理解

> **利用浏览器API窃取保存的凭证**

## 30秒速查卡

| 维度 | 你需要知道的 |
|------|-------------|
| 这是什么？ | 利用浏览器提供的API接口窃取保存在浏览器中的登录凭证 |
| 为什么危险？ | 浏览器保存了大量网站的登录密码，通过API可以直接读取，不需要物理接触用户电脑 |
| 谁需要关心？ | Web安全工程师、SOC分析师、企业安全团队 |
| 你的第一步防御 | 监控浏览器扩展的安装和权限请求，限制浏览器扩展的API访问范围 |
| 如果只做一件事 | 监控Chrome的Login Data和Web Data数据库文件的异常访问，特别是来自非浏览器进程的读取 |

## 难度等级

⭐⭐ 中级 - 需要一定的技术基础和经验

## 这是什么？

**Web凭证API**（T1056.003）是 **输入捕获**（T1056）的一个具体变体，属于 **凭证访问** 阶段的攻击技术。



> 🔍 **打个比方**：就像在浏览器背后偷偷记录你输入的每个网址和密码——攻击者通过API钩子捕获Web凭据输入。

### 具体怎么理解？

利用浏览器API窃取保存的凭证

攻击者使用这种技术时，通常是在凭证访问阶段，想要达到特定的攻击目标。与父技术 T1056 相比，T1056.003 有自己独特的特点和使用场景。

### 为什么有效？

这种技术之所以有效，是因为：
1. **隐蔽性**：利用了正常系统功能或常见协议，不容易被发现
2. **技术门槛适中**：不需要特别高深的技术知识就能实施
3. **广泛适用**：可以在多种环境和系统中使用



## 真实攻击流程

### 典型场景

攻击者在 **凭证访问** 阶段使用 Web凭证API 技术，以下是典型的攻击步骤：

```mermaid
graph TD
    A["入侵目标网站或注入XSS"] --> B["注入恶意JavaScript代码"]
    B --> C["Hook浏览器Credential API"]
    C --> D["拦截密码自动填充数据"]
    D --> E["窃取保存的登录凭证"]
    E --> F["外传至攻击者控制端"]
    style C fill:#ff6b6b,stroke:#333,stroke-width:2px
```

**步骤详解：**

1. **入侵目标网站或注入XSS** - 通过网站漏洞或存储型XSS在目标网页中注入恶意脚本代码
2. **注入恶意JavaScript代码** - 在页面中植入篡改的JS脚本，覆盖或Hook浏览器的Credentials API
3. **Hook浏览器Credential API** - 劫持navigator.credentials.get()等浏览器原生API，在调用时插入窃取逻辑
4. **拦截密码自动填充数据** - 当用户触发密码自动填充时，API调用被重定向，攻击者获取明文凭证
5. **窃取保存的登录凭证** - 从浏览器密码管理器的 autofill 数据中提取用户名和密码

### 主机层检测

**Windows事件ID**：
- 事件ID 4688：进程创建（监控可疑进程）
- 事件ID 4104：PowerShell执行（监控可疑脚本）

**Linux日志**：
- /var/log/auth.log：认证日志
- /var/log/syslog：系统日志

```bash
# 检测异常进程和命令执行
grep -i "suspicious" /var/log/auth.log
```

### 应用层检测


**用人话说：** 这条规则在监控是否有非浏览器进程在读取浏览器保存的密码数据库。Chrome、Firefox等浏览器会把用户保存的密码存在本地数据库中。正常情况下只有浏览器自己会读这些数据库。如果有陌生进程（比如powershell.exe、cmd.exe）试图读取这些文件，那就是攻击者在偷浏览器保存的所有网站密码。

**限制攻击面**：减少暴露的服务和端口，降低被利用的风险

```bash
# 防火墙规则示例：只允许必要的出站连接
```
